## Ключевые компоненты системы

### 1. Конфигурация (`configs/default.yaml`)
Определяет все параметры задачи:
- Размер матрицы `N`
- Метод решения (`bicgstab` или `gmres`)
- Тип предобуславливателя (`ilu0` или `ilut`)
- Точность `tol` и лимит итераций `max_iter`

### 2. Генератор матриц (`generator.py`)
Создаёт тестовые матрицы, имитирующие нефтегазовую модель:
- Разреженная структура с заданной плотностью связей
- Диагональное преобладание для гарантии сходимости
- Вектор правой части с "скважинными" эффектами

### 3. Решатель (`solver.py`)
Реализует два метода:
- **BiCGSTAB**: Стабильный метод сопряжённых градиентов
- **GMRES**: Метод минимальных невязок с перезапуском

Оба поддерживают ILU-предобуславливание для ускорения сходимости.

### 4. Логгер (`metrics.py`)
Фиксирует процесс решения:
- История невязок по итерациям
- Время выполнения
- Экспорт в CSV/PNG/JSON

### 5. Тесты (`test_validation.py`)
Проверяют корректность:
- Сравнение с прямым решателем `spsolve`
- Проверка сходимости при разных параметрах
- Воспроизводимость результатов

## Поток данных

```
YAML конфиг 
    ↓
MatrixGenerator → (A, b)
    ↓
Solver (BiCGSTAB/GMRES + ILU) → (x, info, history)
    ↓
Validator → True/False
    ↓
Logger → CSV + PNG + JSON
```

## Как запустить

```bash
# Запуск с конфигурацией по умолчанию
python main.py

# Запуск с кастомной конфигурацией
python main.py --config configs/my_config.yaml --output-dir results/test1

# Запуск тестов
pytest tests/test_validation.py -v
```

## Ожидаемые результаты

После успешного запуска в папке `results/` появятся:
- `convergence.csv` — таблица итераций с невязками
- `convergence.png` — график сходимости (логарифмическая шкала)
- `summary.json` — краткая статистика (итерации, время, статус)
